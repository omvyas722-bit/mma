Section 0

SYSTEM OVERVIEW

ROAR MMA Command System is an all-in-one staff-facing web application that replaces the patchwork of Evolve Strategies, PGM, spreadsheets, and manual processes currently running the gym. It is a single source of truth for member management, class scheduling, lead tracking, billing, social media analytics, communications, gradings, merch sales, reporting — and is supercharged by a fleet of AI agents that automate as much operational work as physically possible.

The app is staff and coach facing only. Members interact with their existing mobile app. This system runs on a free tech stack — no paid SaaS, no licence fees — and is built to be shipped as a working V1 within one week using Claude Code.

Section 0.2

ROLES & PERMISSIONS

Role Who Access Restrictions
Owner Gym owner Everything. All locations. All data. AI config. Billing setup. Staff management. None
General Manager GM All modules across all locations. Can edit timetable. Can manage leads. Full reports. Can manage all members. Cannot delete staff accounts. Cannot change billing gateway settings.
Front Desk / Admin Reception team Members (view + create + edit), Class bookings, Check-in, Waiver upload, Merch POS, Comms to their location. Basic reports. Cannot see full financial reports. Cannot see AI system. Cannot manage staff.
Coach Coaches (29+) Own class roster and attendance. Own PT client list. Own schedule. View class capacity. Message their class groups. Mark grading eligible. View own PT revenue share. Cannot see other coaches' clients. Cannot see full membership financials. Cannot edit timetable (view only). Cannot access leads.
Sales Con (Salesperson) Full Leads & CRM module. Member view for converting trials. Social media analytics. Lead source reporting. Cannot edit memberships directly. Cannot access billing internals. Cannot access staff management.
Social Media Manager Min Full social media module. Content calendar. Post scheduling. Analytics. Campaign ROI. Cannot see member data. Cannot access billing or leads in detail.

Member Management — Full Interaction Spec

List View
Search & Filter
Search bar does real-time fuzzy search across name, email, phone number. Filter dropdowns: Status (Active, Paused, Cancelled, Trial, Overdue), Type (Ninja 4-6, Kids 6-13, Teen 13-15, Adult, Fighter, Casual), Location (Rocko, Bibra, Both), Martial Art (BJJ, Boxing, MT, Wrestling, Judo, MMA, Kids MMA, Kids Boxing). Filters are additive (AND logic). URL parameters persist filters on refresh. Bulk select with checkboxes allows: bulk message, bulk export, bulk status change.
Real-time: Search updates list as you type (300ms debounce)
Add Member
New Member Form Flow
Multi-step form: Step 1 — Personal details (name, DOB, email, phone, emergency contact). Step 2 — Membership selection (plan, start date, payment method, apply $60 reg fee unless already paid). Step 3 — Waiver & Health declaration (digital sign). Step 4 — Class enrolment (select this week's classes). Step 5 — Confirm & create. System auto-checks if email already exists as a lead and links the records. If DOB makes them under 18, a parent/guardian field is required.
Click: [+ Add Member] button top right
Membership Changes
Plan Change Flow
Clicking [Change Plan] opens a modal. Shows current plan and all other options. User selects new plan, chooses effective date (today, or next billing cycle). System calculates any prorated difference. On confirm: Lightspeed webhook triggered to update direct debit amount. Member receives automated SMS and email confirmation.
Click: [Change Plan] on member profile
Pause Membership
Pause / Hold Flow
Opens modal. Staff enters start date and expected return date (or open-ended). System calculates the $0.71/day hold fee. Staff can override the hold fee amount (Owner only). On confirm: Lightspeed updated, member status changes to "Paused" (orange dot). Member gets SMS/email explaining hold fee and return date. Auto-alert fires 7 days before expected return.
Click: [Pause Membership] on member profile
Trial Members
7-Day Trial Management
Trial members are created when a walk-in or lead books a trial. Trial runs for 7 days, ends on Monday. They can book any class during the trial period. Their profile shows "Trial ends: Mon 15 Apr" in orange. When trial ends, system creates a conversion task for Con (salesperson) and HERMES agent sends a "How was your trial?" email. If they sign up, trial converts to active membership.
Auto: Trial expiry tasks created automatically
Manual: Staff can extend trial from profile
Kids Members
Parent-Child Account Link
Kids (under 18) have their own member profile but are linked to a parent/guardian account. The parent account holds the billing and signed waivers. Searching a parent account shows their linked children. Family discount is applied at billing level: 4th+ child at 50% off. Age categories: Ninjas 4-6 (checked at DOB), Kids 6-13, Teens 13-15. Age categories are enforced at class booking (under-age flagged if booking adult class).
Parent account: Shows "Family" badge and linked kids list
Makeup Classes
Makeup / Catch-Up Booking
Members who missed a class or are returning from a holiday can book a makeup class. Staff creates a makeup booking from the member profile — selects any available class in the current week. This is tracked separately (doesn't count toward their weekly class allocation). Makeup bookings are marked with a "M" indicator in class rosters so coaches can see who is on a makeup.
Click: [Book Makeup Class] on member profile → class picker
Fighter Members
Fighter Account Type
Fighters are standard members but with an "invite" flag. They can attend one extra class per week beyond their plan. Their profile shows: competition history (wins/losses/draws), amateur/pro status, weight class, fight team. Coaches can log fight results here. Fighters are enrolled in specific "Fighters" classes on the timetable which are hidden from non-fighter members in the member app.
Toggle: Fighter flag set by Owner/GM on member profile

Timetable & Classes — Full Interaction Spec

Timetable View
Calendar Navigation & Filters
Default view is the current week. [◀] / [▶] arrows navigate weeks. Location dropdown switches between Rockingham, Bibra Lake, or both (split-screen mode). Filter by martial art, age group, or coach. Each class cell colour codes by fill level: grey (empty/low), amber (50%+), orange (80%+), red (90%+/waitlist). Clicking a cell opens the class detail panel as a right-side drawer.
Click cell: Class detail drawer opens
Right click cell: Quick menu (Cancel / Duplicate)
Add / Edit Class
Class Template Creation
Adding a class creates a template that repeats weekly. Fields: Location, Room (dropdown from configured rooms), Class Name, Martial Art, Level (Fundamentals / Intermediate / Advanced / Kids / Fighter), Age Group, Coach (dropdown), Start Time, Duration, Max Capacity (default 20-30), Recurrence (Weekly, specific days). Setting a minimum belt level locks out under-qualified members from booking in the member app.
Click: [+ Add Class] → modal form
Cancel Class
Class Cancellation Flow
When coach or admin cancels a class: system immediately asks "Notify all booked members?" (default YES). If YES: AI agent HERMES queues SMS and email to all attendees with cancellation notice and an offer to book another class this week. The cancelled class slot shows as grey with a strikethrough on the timetable. Cancellation reason is required (Coach sick / Room unavailable / Low enrollment / Event / Other). Logged for reporting.
Click: [Cancel Class] in class detail panel
Mark Attendance
Attendance / Roll Call
Coach or front desk opens the class, sees all booked members. Each member row has a checkbox. [Mark All Present] pre-checks everyone — coach can then uncheck no-shows. Individual check-in can also be done as members walk in (front desk scans or searches). Attendance is recorded with timestamp. Late check-ins (after class start) are noted. System auto-flags chronic no-shows after 3 missed without cancellation.
Click: Checkbox next to member name in class panel
Capacity Management
Class Cap & Waitlist
When a class hits its cap, new bookings go to a waitlist (ordered by booking time). If a spot opens (cancellation), the first waitlisted member gets an automated SMS/email notification and 2 hours to confirm. After 2 hours, the spot cascades to the next. Staff can also bypass the cap for their location (within reason — recorded as an override). Classes can have a "split room" flag at Bibra Lake to effectively double capacity by using both sides of the mat.
Auto: Waitlist managed automatically by HERMES agent
Edit Template
Bulk Timetable Editing
Owner and GM can open "Edit Template" mode which shows the master weekly template. Changes here propagate to all future weeks. If a class has been edited (e.g. coach swap), a warning shows "This changes all future instances — are you sure?". One-off changes for a specific week (e.g. holiday schedule) use instance-level editing which does NOT affect the template. A visual diff shows what changed before confirming.
Click: [Edit Template] → template mode overlay
Level Requirements
Belt / Skill Gate Enforcement
Classes can have a minimum belt/level requirement: e.g., "BJJ Advanced" requires Blue Belt or above. When a below-level member tries to book via the app, they are blocked with a message. Staff can manually override if the coach has given permission. The system also alerts coaches before class if any unverified or trial members are on the roster for a class with level requirements.
Config: Set on class template — [Minimum Level] field
Fighter Classes
Invite-Only Class Access Control
Fighter classes are hidden from regular members in the app. Only members with the "Fighter" flag on their profile can see and book them. Fighter classes have a minimum attendance requirement — the timetable view for Owner/GM shows a "Fighter Attendance Compliance" badge showing how many fighters attended their required classes this week. Coaches can see who is consistently under-attending and can flag it.
Config: Toggle "Fighters Only" on class template

AI System — Self-Healing

SELF-HEALING & SELF-IMPROVING SYSTEM

HEALER Agent — Self-Healing Loop
  Every 30 seconds, HEALER runs:

  1. CHECK AGENT HEALTH
     For each agent: is last heartbeat < 2 min?
       → NO: restart agent, log incident, alert Owner if critical
       → YES: continue

  2. CHECK API HEALTH
     Test OpenRouter /v1/models endpoint
       → Timeout/Error: switch model to backup (openrouter/free fallback)
       → Log degradation window

  3. REVIEW FAILED TASKS (from last hour)
     For each failed task in logs:
       → Retry up to 3 times with exponential backoff
       → If all retries fail: escalate to human (notification)

  4. ANALYSE REJECTION PATTERNS (daily)
     Review human-rejected AI messages from last 7 days
       → If same type of message rejected 3+ times:
         Generate a prompt improvement suggestion
         Surface to Owner: "HEALER suggests: modify HERMES prompt
         for lead follow-ups to be less sales-y"
         Owner approves → system prompt updated

  5. SELF-IMPROVEMENT (weekly)
     Generate report: "These agent tasks failed most often this week"
     Propose 3 specific prompt or workflow improvements
     Surface in Mission Control for Owner review
🔬 Self-Improving Loop HEALER analyses rejection patterns and failure logs weekly, then proposes concrete system prompt edits. These proposals appear in Mission Control for the Owner to approve or reject. Approved changes are applied to the relevant agent's system prompt immediately. This creates a feedback loop where the AI agents gradually get better calibrated to ROAR MMA's specific tone and member base over time.
